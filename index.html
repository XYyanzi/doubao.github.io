获取java程序的堆栈文件并使用VisualVM进行分析

发表于 2021-07-17 分类于 Java

docker内运行的程序导出堆栈文件方法:
1、进入docker容器内部
docker ps 查到对应的容器id
docker exec -it 容器id /bin/bash
2、在容器内部执行命令，查找程序运行的pid
jps -l
3、在容器内执行命令，生成堆栈文件
jmap -dump:live,format=b,file=heap001 1
其中最后的1 是上一步查出来的pid
4、退出容器，将容器内文件导出到宿主机服务器上
在容器外执行命令
docker cp 容器id:文件路径 宿主机路径
docker cp e79d237f703d:/app/heap001.dump /data
5、查看导出的文件
echo $JAVA_HOME 查找java_home路径并进入
执行 jvisualvm 命令 启动软件
打开/heap001.dump文件

异常

异常: 指的是程序在执行过程中，出现的非正常的情况，最终会导致JVM的非正常停止。

Throwable类是Java语言中所有错误或异常的超类。两个子类: Error\Execption

Exception(异常): 正确处理后可继续执行
- 编译器异常: 编译java时出现问题
- 运行期异常(RuntimeExecption): 程序运行期间出现问题
Error(错误):
- 程序得了无法治愈的的病，必须修改源代码才能继续执行

处理异常:
- 直接throws到上层方法，最终终止程序
- try…catch..

异常处理的五个关键字: try、catch、finally、throw、throws

抛出异常Throw

可以使用throw关键字在指定的方法中抛出指定的异常。

  • throw关键字必须写在方法的内部
  • throw关键字后边new的对象必须是Exception或者Exception的子类对象
  • throw关键字抛出指定的异常对象，我们就必须处理这个异常对象
    • 如果throw的是RuntimeException或者RuntimeException的子类，可以不处理，默认交给JVM处理
    • 如果throw的是编译异常，就必须处理这个异常，要么throws要么try chatch

声明异常thorws

异常处理的第一种方式:交给别人处理。会把异常对象声明抛出给方法的调用者处理。在方法声明时使用。

  • throws关键字必须卸载方法声明处
  • throws关键字后面声明的异常必须是Exception或者Exception的子类
  • 方法内部如果抛出了福讴歌异常对象，namethrows后边必须也声明多个异常，如果抛出的多个异常对象有子父类关系，那么直接声明父类异常即可
  • 调用了一个声明抛出异常的方法，我们就必须处理声明的异常，要么继续throws抛出，要么自己try…catch..

try…catch

异常处理的第二种方式:自己处理异常

  • try中可能会抛出多个异常对象，可以使用多个catch处理这些异常对象
  • 如果try中产生了异常，就会执行catch中的异常处理逻辑，执行完毕后会继续执行之后的代码，如果try中没有产生异常就不会执行catch中的异常处理逻辑，直接执行后面的代码

finally代码块

try…catch…finally: 无论是否发生异常都需要执行的代码需要放到finally代码块中。比如在try语句中打开了一些物理资源，最终都需要关闭，可以在finally代码块中处理关闭资源的代码。

  • finally不能单独使用，必须和try一起使用
  • finally一般用于资源回收，无论程序会否出现异常最后资源都要释放
  • finally语句中避免出现return语句

多线程

并发: 指两个或多个时间在同一时间段内发生(一个cpu交替执行)
并行: 指两个或多个时间在同一时刻发生(同时发生)(两个cpu同时)

进程: 内存中运行的一个应用程序，每个进程拥有独立的内存空间
线程: 线程是进程中的一个执行单元，一个进程中至少有一个线程。
一个程序运营后至少有一个进程，一个进程可以包含多个线程。

主线程

主线程: 执行主方法(main)的线程。
执行从main方法开始，从上到下依次执行

多线程原理

创建线程的两种方式

1、创建Thread类的子类，重写Thread类的run方法，然后使用子类实例调用start方法开启多个线程
2、实现Runnable接口并重写其中的run方法，之后创建Runnable实现类的实例，并以此实例作为Thread的target来创建Thread对象，该Thread对象才是真正的线程对象，调用线程对象的start方法启动线程

实现Runnable接口比集成Thread类所具有的优势

1、避免了单继承的局限性
一个类只能继承一个类，继承了Thread类之后就不能继承其他类，而实现Runnable接口的方式还可以继承其他类和实现其他接口
2、增强了程序的扩展性，降低了程序的耦合性
实现Runnable接口的方式把设置线程任务(创建实现类对象)和开启新线程(创建Thread类对象调用start方法)进行了解耦
3、线程池只能放入实现Runnable或Callable类线程，不能直接放入集成Thread的类

解决线程安全问题的三种方式:
1、同步代码块
2、同步方法
3、Lock锁机制

数据结构

数据存储的常用结构有: 栈、队列、数组、链表、红黑树集合

栈(stack)特点:先进后出

队列(queue)特点: 先进先出

数组(Array)特点: 查询快，增删慢

查询快: 数组的地址是连续的，我们可以通过数组的首地址找到数组，然后通过数组的素银可以快速查找某个元素
增删慢: 数组的长度是固定的，如果要增加或删除一个元素，必须创建一个新数组，把源数组的数据复制过来

链表(linked list)特点: 查询慢，增删快

链表是由一系列节点(Node)组成,节点可以在运行时动态生成。每个节点包括两部分:一个数据域和两个指针域。

指针域 数据域 指针域
自己的地址 数据 下一个节点的地址

查询慢: 链表中的地址不是连续的，每次查询元素都必须从头开始
增删快: 链表结构增加或删除一个元素，对链表的整体结构没有影响，所以增删快

链表有两种类型: 单向链表和双向链表

单向链表: 链表中只有一条链子，不能保证元素的顺序
双向链表: 链表中有两条链子，有一条链子是专门记录元素的顺序的。因此双向链表是一个有序集合。

二叉树(binary tree): 每个节点不超过2个的有序树

排序树(查找树): 在二叉树的基础上，元素是有大小顺序的。左子树小，右子树大。

平衡树: 左子树和右子树相等

红黑树: 查询速度非常快感。趋近于平衡树，查询的速度非常快。查询叶子节点最大次数和最小次数不能超过2倍。

List接口

特点: 有序、有索引(可以使用带索引的方法)、允许有重复的元素

ArrayList集合

ArrayList底层是个数组结构，但长度可变。并且多线程实现。查询快，增删慢。

LinkedList集合

LinkedList集合存储数据的结构是链表结构（双向链表）。查询慢、增删快。包含了大量操作首尾元素的方法。

Vector集合

和ArraryList一样底层存储数据的结构是数组，并且是单线程。java1.2版本后被List集合代替。

Set接口

特点: 无序、没有索引，不能存储重复数据

Set集合取出元素可以使用迭代器和增强for循环

HashSet集合

HashSet特点:存储数据的接口是哈希表结构，查询速度快;无序的；单线程的。

哈希表

哈希表: 在JDK1.8之前，哈希表底层采用数组+链表实现, 即使用链表来处理hash值冲突。JDK1.8之后，哈希表是由数组+链表+红黑树实现的。查询速度很快。

哈希值: 是一个十进制的整数，由系统随机给出(就是对象的地址值，是一个逻辑地址，是模拟出来的地址，不是数据实际存储的物理地址值)
在Object类中有一个方法可以获取对象的哈希值hashCode()方法
String类重写了hashCode()方法，如果字符串内容相同，则hashCode(相同)

哈希冲突: 对象内容不同，但对应的hashCode()是相同的。例如: ”重地“ 和 ”通话“

LinkedHashSet集合

继承了和HashSet集合，但是是有序的。
底层是一个哈希表(数组+链表/红黑树) + 链表,多了一个链表来记录元素的存储顺序，保证元素有序。

Map集合

HashMap

特点:

1、Map集合是一个双列集合，一个元素包含两个值(一个key，一个value)
2、Map集合中的元素，key和value的数据类型可以相同，也可以不同
3、Map集合中的元素，key是不允许重复的，value可以重复
4、Map集合中的元素，key和value是一一对应的，一个key对应一个value

常用方法:

public V put(K key, Vvalue): 把指定的键与指定的值添加到map集合中。返回值V: 存储键值对的时候，key不重复，返回值V是null，如果key重复，会使用新的value替换map中重复的value,返回被替换的value的值

public V remove(Object key): 把指定的键 多对应的键值对元素在集合中删除，返回被删除元素的值。返回值V: key存在，返回被删除的值，key不存在返回null

public V get(Object key): 通过指定的键，获取对应的值。返回值V: key存在，返回对应的value的值，key不存在返回null

boolean containsKey(Object key): 判断集合中时候包含指定的key。key存在，返回true，key不存在返回false

遍历map集合:

1、keySet(): 把集合中的所有key取出来存放在set集合中，然后编辑set，并且通过key取出所有的value
2、Entry(): map集合中的一对对的键值对又叫做Entry对象。Entry将键值对的对应关系封装成了对象。

LinkedHashMap

继承了HashMap，底层结构多了一个链表用来存储元素的顺序，因此具有可预知的迭代顺序。（存储和取出顺序一致）

Hashtable

和HashMap的区别:
1、 key和value都不允许为空
2、 是一个线程安全的集合，是单线程集合，速度慢

Hashtable和Vector集合一样，在JDK1.2版本之后被更先进的集合(HashMap,ArrayList)取代了
Hashtable的子类Properties依然活跃
Properties集合是一个唯一和IO流相结合的集合

Collection集合

集合就是一种容器，可以用来存储多个数据。
集合和数组的区别:

  • 数组的长度的固定的，集合的长度是可变的。
  • 数组可以存储基本数据类型也可以存储对象，但必须是同一类型的元素。集合只能存储对象，而且对象的类型可以不一致。

Iterator迭代器

迭代: 是Collection集合元素的通用获取方式。在取出元素之前先判断集合中是否有元素，如果有就把这个元素取出来，然后继续在判断，直到把集合中元素全部取出。这种取出方式的专业术语就叫迭代。

Iterator接口的两个常用方法
boolean hasNext(E e) 判断集合中是否存在元素
E next() 取出迭代的下个元素

Iterator是一个接口，不能直接创建对象，需要通过实现类来创建对象，它获取实现类对象的方式比较特殊。Collection接口中有个iterator()的方法，这个方法返回的就是Iterator的实现类对象。

迭代器的使用步骤:

  • 使用集合中的iterator()方法获取迭代器的实现类对象，并使用Iterator接口来接收(多态)
  • 使用Iterator接口中的方法HasNext()判断集合中是否存在元素
  • 使用Iterator接口中的方法Next()取出集合中的下一个元素

使用iterator()方法创建实现类对象时，会创建一个指向-1索引的头指针
hasNext()方法用来判断当前指针后面是否还有元素存在
next()方法做了两件事，一是取出下一个元素的值，二是将指针向后移动一位

JDK1.5之后出现的新特性，简化迭代器的for循环书写

final关键字

final关键字代表最终的、不可改变的。

可以修饰一个类: 被修饰的类不能有任何子类
可以修饰一个方法: 被修饰的方法不能被子类覆盖重写

2、间接通过成员方法访问成员变量:

方法属于谁就优先用谁，没有则向上找

子类中的三种变量访问方式:
局部变量 直接写变量名
本类的成员变量 this.变量名
父类的成员变量 super.变量名

方法重写

重写的概念: 在继承关系中，方法名称和参数列表都一样，叫做重写(OverWrite)

重写(OverWrite): 方法名称相同，参数列表也相同
重载(OverLoad): 方法名称相同，参数列表不同

注意事项:

1、必须保证父子类之间方法的名称、参数列表都相同
2、子类方法的返回值范围必须小于等于父类方法的返回值范围
3、子类方法的权限必须大于等于父类方法的权限修饰符

public > protected > (default) > private

继承关系中，父子类构造方法的访问特点:
1、子类构造方法中有一个默认隐含的”super()”调用，因此创建子类对象时，一定是先执行父类的构造方法，再执行子类的构造方法
2、子类构造可以通过super关键字来调用父类重载构造方法
3、super()的父类构造调用，必须是子类构造方法的第一个语句。一个子类只能调用一次super()方法

总结:子类必须调用父类的构造方法，不写则赠送super()，写了就用指定的super调用。super()只能有一个且只能在子类构造方法的第一行

super关键字的三种用法:
1、子类成员方法访问父类的成员变量
2、子类的成员方法访问父类的成员方法
3、子类的构造方法访问父类的构造方法

this关键字的三种用法:
1、本类的成员方法中访问本类的成员变量
2、本类的成员方法中访问本类的另一个成员方法
3、本类的构造方法中访问本类的另一个构造方法

super和this两种构造调用，不能同时使用

java语言是单继承的
java语言可以多级继承
一个子类的直接父类是唯一的，但是一个父类可以拥有多个子类

抽象的概念:
如果父类当中的方法不确定方法体的具体实现，那么这就应该是一个抽象方法。

抽象类和抽象方法的使用方法:
1、不能直接创建抽象类对象，必须使用一个子类来继承抽象类
2、子类必须覆盖重写父类中所有的抽象方法。（也就是实现）

接口

生活中: 接口就是一种公共的规范标准。
程序中: 接口就是多个类的公共规范。
接口是一种引用类型，最重要的内容就是其中的抽象类方法。

java7的接口内容包括: 常量、抽象方法
java8的接口内容包括: 常量、抽象方法、默认方法、静态方法
java9的接口内容包括: 常量、抽象方法、默认方法、静态方法、私有方法

接口的使用:
1、接口不能直接使用，必须有一个”实现类”来实现该接口
public class 实现类名称 implements 接口名称{}
2、接口的实现类必须覆盖重写接口的所有抽象类
3、创建实现类的对象进行使用

接口的默认方法:
默认方法有方法体，主要为了解决接口升级的问题.
接口的默认方法可以通过接口实现类对象直接调用
接口的默认方法也可以被接口实现类覆盖重写

接口的静态方法:
不能通过实现类对象来调用，只能通过接口名称直接调用 接口名.静态方法()

接口的私有方法:
接口的私有方法只有接口内部可以调用，实现类不可以调用。(主要用于接口内部的静态方法和默认方法调用)
普通私有方法: 提供给接口默认方法调用
静态私有方法: 提供给接口静态方法调用

接口常量的定义和使用:
接口当中的常量: public static final 数据类型 常量名称 = 数据值; 其中（public static final）可以不写, 即使不写也是常量，不可以修改。

类与类之间是单继承的。直接父类只有一个。
类与接口是多实现的。一个类可以实现多个接口。
接口与接口之间是多继承的。

多态

【对象的多态性】:一个对象拥有多个形态，就是多态。

小明是一个学生，但同时也是一个人。
小明是一个对象，这个对象既有学生形态，也有人类形态。

使用格式

【父类引用指向子类对象】
父类名称 对象名 = new 子类名称()
接口名称 对象名 = new 实现类名称()

1、直接通过对象名称访问: 看创建对象时=左边是谁，优先用谁，没有则向上找
2、间接通过成员方法访问: 看new出来的是谁，就运行谁，没有则向上找

成员变量: 编译看左边，运行也看左边
成员方法: 编译看左边，运行看右边

对象的向上转型其实就是多态的写法: 父类名称 对象名 = new 子类名称();
含义： 右侧创建一个子类翠香，把他当做父类来看待
向上转型一定是安全的，因为是从小范围转向了大范围

含义: 将父类对象还原为本来的子类对象

字符串

字符串的特点:
1、字符串的内容永不可变,是一个常量
2、正是因为字符串不可改变，所以字符串是可以共享使用的。
3、字符串效果上相当远char[]字符数组，但是底层原理是byte[]字节数组

创建字符串的方式

三种构造方法，一种直接创建

  • public String(): 创建一个空白字符，不含任何内容
  • public String(char[] array): 根据字符数组的内容来创建字符串
  • public String(byte[] array): 根据字节数组的内容来创建字符串
  • String str = “hello”; 右边直接用双引号

注意:直接写上双引号，就是字符串对象

字符串常量池: 程序中直接写双引号创建的字符串，就在字符串常量池中。 【堆内存中保存, 实际保存的是字节数组的地址, new 创建的字符串不再常量池中。
==判断: 对于基本数据类型，比较的是值，对于引用类型，比较的是地址值

equals()方法

如果要判断两个引用对象的值是否相等，可以使用equals方法:
public boolean equals(Object obj)
注意事项:
1、equals方法具有对称性，a.equals(b) 和b.equals(a)是一样的
2、如果比较的是一个常量和一个变量，推荐将常量写在前面 “q”.equals(b),防止变量的值是null的情况下报空指针错误.
3、equalsIgnoreCase() 方法可以忽略英文大小写

static关键字

被static关键字修饰的成员变量或成员方法，不属于对象自己，而是属于类，本类对象共享。

注意事项:
1、静态不能直接访问非静态
原因：在内存当中【先】有静态内容，【后】有的非静态
2、静态方法中不能使用this
原因：this是只当前对象，但静态方法不需要对象
3、静态优先于非静态，因此静态代码块比构造函数先执行

静态代码块的典型用途: 用来一次性的对静态变量进行赋值

静态变量和静态方法在内存中是在方法区的静态区中。

类和对象

面向对象三大特征:

  • 封装
  • 继承
  • 多态

类: 是一组相关属性和行为的集合。可以看成是一类事物的模板，使用事物的属性特征和行为来描述该类事物
对象：是一类事物的具体实现。对象是类的一个实例，必然具备该类事物的属性和行为。

类是对一类事物的描述，是抽象的。
对象是一类事物的实例，是具体的。
类是对象的模板，对象是类的实体。

类的定义

类的使用

通常情况下，一个类并不能直接使用，需要根据类创建一个对象才能使用。

  • 导包
    • 指出需要使用的类的所在位置 import 包名称.类名称
    • 如果和要使用的类在相同的包下，可以胜率导包语句
  • 创建
    • 类名称 对象名 = new 类名称()
    • Student stu = new Student()
  • 使用
    • 使用成员变量 对象名.成员变量
    • 使用成员方法 对象名.成员方法(参数)

如果成员变量没有进行赋值，name将会有一个默认值，规则和数组一样。

new出来的对象包括成员变量和成员方法都是存储在堆内存中
堆内存中存储的是成员方法在方法区的地址值
执行成员方法时，也会进入到栈内存中，成员方法执行结束后会立即出栈


成员变量和局部变量的区别

  • 定义的位置不一样
    • 局部变量: 方法内部
    • 成员变量: 方法外部，直接写在类里
  • 作用范围不一样
    • 局部变量: 只有方法中可以使用，出了方法就不能再使用
    • 成员变量: 整个类都可以使用
  • 默认值不一样
    • 局部变量: 没有默认值，需要手动进行赋值
    • 成员变量: 有默认值，规则和数据一样
  • 内存位置不一样
    • 局部变量: 栈内存
    • 成员变量: 堆内存
  • 生命周期不一样
    • 局部变量: 随着方法进栈而诞生，随着方法出栈而消失
    • 成员变量: 随着对象创建而诞生，随着对象被垃圾回收而消失

封装性

封装性在java中的体现
1、方法就是一种封装
2、关键字private也是一种封装

private关键字修饰需要保护的成员变量。被private修饰的成员变量，只能在本类中进行访问，超出了本类范围之外就不能再直接访问(类名.成员变量的方式)了。

本类范围外想访问被private修饰的成员变量: 通过getter、setter方法
对于基本类型boolean类型的成员变量，在getter时，方法名应该是isXx()

this关键字: 当方法的局部变量和类的成员变量重名的时候，根据”就近原则”，优先使用局部变量，如果需要访问本类当中的成员变量，需要使用this关键字
通过谁调用的方法，谁就是this，this就代表类本身

构造方法

构造方法是专门用来创建对象的方法,当我们通过关键字new来创建对象时，其实就是在调用构造方法。 （new对象的时候实际就是在调用构造方法）

  • 构造方法的名称和类名称必须一样，大小写也要一样
  • 构造方法不能写返回值类型
  • 构造方法不能return一个具体的返回值
  • 如果没有编写任何构造方法，那么编译器会默认创建一个空的构造方法
  • 一旦自己编写了至少一个构造方法，编译器就不会默认创建构造方法了

一个标准的类的组成:

  • 所有的成员变量都要使用private关键字进行修饰
  • 为每一个成员变量编写一对Getter/Setter方法
  • 编写一个无参数的构造方法
  • 编写一个全参数的构造方法

这样标准的类也叫做Java Bean

数组

数组的定义

数组: 是一种容器，可以同时存放多个数据值，存放的数据类型必须统一
数组的特点:
  1. 数组是一种引用数据类型
  2. 数组当中的数据，类型必须统一
  3. 数组的长度在程序运行期间不可以改变

数组的初始化: 在内存中创建一个数组，并且向其中赋予一些默认值
两种初始化方式:
  1.动态初始化(指定长度)
  2.静态初始化(指定内容)

动态初始化格式:
数据类型[] 数组变量名称 = new 数据类型[数组长度]

静态初始化格式:
数据类型[] 数组变量名 = new 数据类型[]{元素1，元素2，…};
数据类型[] 数据变量名 = {元素1，元素2，…};

数组的使用

获取数组中的值: 数组名[索引值]
索引值从0开始，到“数组长度-1”为止

注意事项：

使用动态初始化数组时，其中的元素会自动拥有一个默认值

  • 如果是整数类型，默认0
  • 如果是浮点类型，默认0.0
  • 如果是字符类型，默认‘\u0000’
  • 如果是布尔类型，默认是false
  • 如果是引用类型，默认是null

java内存

Java的内存需要划分成为5个部分:

  • 栈内存(Stack): 存放的都是方法中的局部变量。方法的运行一定要在栈当中。

    局部变量: 方法的参数，或者是方法()内部的变量，一旦超出作用域，立即从栈内存中消失

  • 堆内存(Heap): 凡是New出来的东西，都在堆内存中。

    堆内存里面的东西都有一个地址值: 16进制
    堆内存里面的数据都有默认值

  • 方法区(Method Area): 存储.class相关信息，包含方法的信息。

  • 本地方法栈(Native Method Stack): 与操作系统相关。

  • 寄存器(pc Register): 与CPU相关。

数组的内存图

1、运行.class文件时，先从方法区中获取class信息
2、读取到main方法后，会在栈中开辟一个空间，供main方法运行，这个过程叫做进栈
3、定义一个变量array，就是在栈中创建一个局部变量
4、new数组的时候，就是在堆内存中开辟1块空间，该空间有对应的地址值，该地址值会赋值给变量。同时每个数组索引对应的空间中存放默认值
5、输出变量array时，实际输出的是数组的地址值
6、给array[1]赋值时，实际是将原数组索引1的位置的值由默认值替换成实际值
