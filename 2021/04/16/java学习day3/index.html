IDEA常用快捷键(MAC)

Cmd+D 复制光标所在行，插入在光标位置下面
Cmd+Option+L 格式化代码
Cmd+/ 单行注释
Ctrl+Shift+/ 多行注释
Cmd+N 自动生成代码, toString，get，set等方法
Option+Shift+上下箭头 移动当前代码行

方法

方法其实就是若干语句的功能集合

方法的定义格式

修饰符 返回值类型 方法名(参数类型 参数名,…参数类型 参数名){
  方法体
  return 返回值;
}

方法的调用流程图解

  • 【1】找到方法定义
  • 【2】传递参数
  • 【3】执行方法体
  • 【4】带着返回值回到方法调用处

方法的重载

方法重载(OverLoad)的概念:
多个方法的名称相同,但参数列表不一样。(参数个数、类型、多类型顺序不同), 与方法的返回值类型和方法修饰符无关

数组

数组的定义

数组: 是一种容器，可以同时存放多个数据值，存放的数据类型必须统一
数组的特点:
  1. 数组是一种引用数据类型
  2. 数组当中的数据，类型必须统一
  3. 数组的长度在程序运行期间不可以改变

数组的初始化: 在内存中创建一个数组，并且向其中赋予一些默认值
两种初始化方式:
  1.动态初始化(指定长度)
  2.静态初始化(指定内容)

动态初始化格式:
数据类型[] 数组变量名称 = new 数据类型[数组长度]

静态初始化格式:
数据类型[] 数组变量名 = new 数据类型[]{元素1，元素2，…};
数据类型[] 数据变量名 = {元素1，元素2，…};

数组的使用

获取数组中的值: 数组名[索引值]
索引值从0开始，到“数组长度-1”为止

注意事项：

使用动态初始化数组时，其中的元素会自动拥有一个默认值

  • 如果是整数类型，默认0
  • 如果是浮点类型，默认0.0
  • 如果是字符类型，默认‘\u0000’
  • 如果是布尔类型，默认是false
  • 如果是引用类型，默认是null

java内存

Java的内存需要划分成为5个部分:

  • 栈内存(Stack): 存放的都是方法中的局部变量。方法的运行一定要在栈当中。

    局部变量: 方法的参数，或者是方法()内部的变量，一旦超出作用域，立即从栈内存中消失

  • 堆内存(Heap): 凡是New出来的东西，都在堆内存中。

    堆内存里面的东西都有一个地址值: 16进制
    堆内存里面的数据都有默认值

  • 方法区(Method Area): 存储.class相关信息，包含方法的信息。

  • 本地方法栈(Native Method Stack): 与操作系统相关。

  • 寄存器(pc Register): 与CPU相关。

数组的内存图

1、运行.class文件时，先从方法区中获取class信息
2、读取到main方法后，会在栈中开辟一个空间，供main方法运行，这个过程叫做进栈
3、定义一个变量array，就是在栈中创建一个局部变量
4、new数组的时候，就是在堆内存中开辟1块空间，该空间有对应的地址值，该地址值会赋值给变量。同时每个数组索引对应的空间中存放默认值
5、输出变量array时，实际输出的是数组的地址值
6、给array[1]赋值时，实际是将原数组索引1的位置的值由默认值替换成实际值
