类和对象

面向对象三大特征:

  • 封装
  • 继承
  • 多态

类: 是一组相关属性和行为的集合。可以看成是一类事物的模板，使用事物的属性特征和行为来描述该类事物
对象：是一类事物的具体实现。对象是类的一个实例，必然具备该类事物的属性和行为。

类是对一类事物的描述，是抽象的。
对象是一类事物的实例，是具体的。
类是对象的模板，对象是类的实体。

类的定义

类的使用

通常情况下，一个类并不能直接使用，需要根据类创建一个对象才能使用。

  • 导包
    • 指出需要使用的类的所在位置 import 包名称.类名称
    • 如果和要使用的类在相同的包下，可以胜率导包语句
  • 创建
    • 类名称 对象名 = new 类名称()
    • Student stu = new Student()
  • 使用
    • 使用成员变量 对象名.成员变量
    • 使用成员方法 对象名.成员方法(参数)

如果成员变量没有进行赋值，name将会有一个默认值，规则和数组一样。

new出来的对象包括成员变量和成员方法都是存储在堆内存中
堆内存中存储的是成员方法在方法区的地址值
执行成员方法时，也会进入到栈内存中，成员方法执行结束后会立即出栈


成员变量和局部变量的区别

  • 定义的位置不一样
    • 局部变量: 方法内部
    • 成员变量: 方法外部，直接写在类里
  • 作用范围不一样
    • 局部变量: 只有方法中可以使用，出了方法就不能再使用
    • 成员变量: 整个类都可以使用
  • 默认值不一样
    • 局部变量: 没有默认值，需要手动进行赋值
    • 成员变量: 有默认值，规则和数据一样
  • 内存位置不一样
    • 局部变量: 栈内存
    • 成员变量: 堆内存
  • 生命周期不一样
    • 局部变量: 随着方法进栈而诞生，随着方法出栈而消失
    • 成员变量: 随着对象创建而诞生，随着对象被垃圾回收而消失

封装性

封装性在java中的体现
1、方法就是一种封装
2、关键字private也是一种封装

private关键字修饰需要保护的成员变量。被private修饰的成员变量，只能在本类中进行访问，超出了本类范围之外就不能再直接访问(类名.成员变量的方式)了。

本类范围外想访问被private修饰的成员变量: 通过getter、setter方法
对于基本类型boolean类型的成员变量，在getter时，方法名应该是isXx()

this关键字: 当方法的局部变量和类的成员变量重名的时候，根据”就近原则”，优先使用局部变量，如果需要访问本类当中的成员变量，需要使用this关键字
通过谁调用的方法，谁就是this，this就代表类本身

构造方法

构造方法是专门用来创建对象的方法,当我们通过关键字new来创建对象时，其实就是在调用构造方法。 （new对象的时候实际就是在调用构造方法）

  • 构造方法的名称和类名称必须一样，大小写也要一样
  • 构造方法不能写返回值类型
  • 构造方法不能return一个具体的返回值
  • 如果没有编写任何构造方法，那么编译器会默认创建一个空的构造方法
  • 一旦自己编写了至少一个构造方法，编译器就不会默认创建构造方法了

一个标准的类的组成:

  • 所有的成员变量都要使用private关键字进行修饰
  • 为每一个成员变量编写一对Getter/Setter方法
  • 编写一个无参数的构造方法
  • 编写一个全参数的构造方法

这样标准的类也叫做Java Bean
