selenium介绍

Selenium is a suite of tools to automate web browsers across many platforms.
selenium是一个可以在不同平台实现浏览器自动化的一套工具
主要由三个工具组成: WebDriver、IDE、Grid

  • runs in many browsers and operating systems 可以在多种浏览器和操作系统中运行
  • can be controlled by many programming languages and testing frameworks 支持多种程序语言和测试框架 官网

三种等待方式

  • 直接等待

    强制等待，线程休眠一段时间 Thread.sleep(5)

  • 隐式等待

    设置一个等待时间，轮询查找(默认时间0.5s)元素是否出现，如果不出现抛出异常 dirver.manage().timeout().implicitlyWait(5, TimeUnit.SECONDS)

  • 显示等待

    程序每隔一段时间(默认时间0.5s)进行条件判断，如果条件成立则继续执行，否则继续等待，直到超时。 通过WebDriverWait结合until()方法实现

隐式等待和显式等待的区别:
隐式等待用于全局设置，通常在setup中定义，在之后所有的查找元素时都进行隐式等待。显示等待可以针对某单个元素进行设置。

web控件定位与常见操作

XPath和CSS Selector

XPath是XML Path language, 用于解析HTML和XML。 速度比css selector慢很多，原因是查找元素是从头到尾的遍历查找。但是appium中原生控件不支持css selector，只能用xpath。

p:nth-child(2) 表示p元素对应的父元素的第2个子元素
xpath中的/代表子节点，//代表子孙节点
css selector中的 空格代表子节点 >代表子孙节点

$('#navigation-bar li:nth-child(4)')
$x('//*[@id="navigation-bar"]//li[4]')

控件操作进阶

Actions类是selenium对键盘、鼠标进行操作的类
执行原理：
调用Actions方法时不会立即执行，而是将所有的操作按顺序存放在一个队列里，当你调用perform()方法时，队列中的事件会一次执行。
基本用法：
生成一个动作 Actions action = new Actions(driver)
动作添加方法1 action.方法1
动作添加方法2 action.方法2
调用perform()方法执行 action.perform()

.click() 鼠标左键点击
.sendKeys() 键盘输入

.doubleClick() 鼠标左键双击
.contextClick() 鼠标右键单击
.moveToElement() 将鼠标移动到某个元素上
.gradAndDrop(被拖拽元素,目标元素) 将元素拖拽到另一个元素位置
.keyDown(键盘按键) 按下键盘的某个键
.keyUp(键盘按键) 抬起键盘的某个键

多窗口处理与网页frame

多窗口切换

多窗口切换通过切换句柄实现
1、先获取到当前的窗口句柄 driver.getWindowHandle()
2、再获取到所有的窗口句柄 driver.getWindowHandles()
3、判断是否是想要操作的窗口，如果是，继续操作，如果不是，进行切换之后再操作 driver.switchTo().window(name)

frame切换

web自动化中如果一个元素定位不到，name很大可能是在iframe中。
frame是html中的框架，在html中所谓的框架就是可以再同一个浏览器中显示不止一个页面。
frame分为 frameset、frame、iframe三种
frame和iframe对selenium定位而言是不样的，selenium有一组专门的api对frame进行操作。
frame存在两种，一种是嵌套的，一种是未嵌套的。

切换frame
driver.switchTo().frame() 根据元素id或者index切换frame
driver.switchTo().defaultContent() 切换到默认frame
driver.switchTo().parentFrame() 切换到父级frame

对于未嵌套的iframe可以直接使用id进行定位，也可以使用index进行定位
对于嵌套的iframe先进入到iframe的父节点，再进到子节点，然后可以对子节点里面的对象进行处理和操作
